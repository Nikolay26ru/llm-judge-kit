# LLMJudge Kit v0.1.2

A polish release: more built-in rubrics, a frozen + documented public API, a
matured repository, and runnable framework integrations. No breaking changes.

```bash
pip install --upgrade llm-judge-kit
```

## Added

- **Two new built-in rubrics** — `coherence` (logical organization / internal
  consistency) and `completeness` (covers every requested part). Seven built-ins now.
- **CLI `--version`** — `llm-judge-kit --version`.
- **Frozen, documented public API** — [`docs/api.md`](../api.md) describes the full
  surface, and a snapshot test makes it impossible to change `llm_judge_kit.__all__`
  by accident.
- **Runnable integration examples** ([`examples/integrations/`](../../examples/integrations/))
  for LangChain, LlamaIndex, DSPy, and a raw pipeline — each runs offline and is
  exercised in CI.
- **Mature-OSS repo** — `SECURITY.md`, `CODE_OF_CONDUCT.md`, issue/PR templates,
  PyPI + Python-version badges, and a "use it as a CI gate" recipe in the README.

## Quality

- `mypy --strict` clean, ships `py.typed`; ~99.9% coverage; CI on Python 3.11/3.12/3.13.
- Core still has **zero required runtime dependencies**; provider SDKs stay behind
  optional extras.

See [`CHANGELOG.md`](../../CHANGELOG.md) for the full list.

# LLMJudge Kit v0.1.0

First public release — a provider-agnostic, reproducible, typed
**LLM-as-a-judge** primitive with a zero-dependency core.

```bash
pip install llm-judge-kit
```

## Highlights

- **`Judge` core** — pair a provider with a rubric; `score()` returns a typed,
  frozen `JudgeResult` that casts (`float(r)`) and compares (`r > 0.8`) like its
  score, with `reason`, `evidence`, `violations`, and `metadata`.
- **Providers** — OpenAI-compatible, Anthropic, and Ollama, plus a deterministic
  offline `MockProvider`. Provider SDKs are imported lazily and live behind
  optional extras, so the core stays dependency-free.
- **Consensus & reliability** — `ConsensusJudge` / `Judge.consensus(...)` (confidence
  derived from agreement), composable `RetryProvider` and `CachingProvider`.
- **pytest plugin** — the `llm_judge_kit` fixture and `assert_passes` turn evals
  into ordinary tests; failures show score, reason, and violations.
- **CLI & reporting** — `llm-judge-kit eval / compare / report` with
  JSON / Markdown / HTML output and a `--fail-under` gate for CI; a small
  benchmark engine and `.jsonl`/`.json` dataset loader.

## Quality

- `mypy --strict` clean, ships `py.typed`.
- ~99.9% test coverage; unit tests run fully offline (no network).
- CI green on Python 3.11 / 3.12 / 3.13.
